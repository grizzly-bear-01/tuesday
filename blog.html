What is the difference in map, filter, forEach, and find?

All four functions - map, filter, forEach, and find - are built-in methods in JavaScript that can be used with arrays to perform various operations on their elements. Here are the differences between them:

map: map method creates a new array by applying a function to each element of the original array. The function returns a new value, and that value is used to create a new array. The length of the new array will always be the same as the original array.
Example: 

const arr = [1, 2, 3, 4, 5];
          const newArr = arr.map((num) => num * 2);
          console.log(newArr); 

          // [2, 4, 6, 8, 10]

filter: filter method creates a new array by filtering out the elements that do not meet the criteria specified in a function. The function returns a Boolean value, and only the elements for which the function returns true are included in the new array.
Example: 

const arr = [1, 2, 3, 4, 5];
          const newArr = arr.filter((num) => num % 2 === 0);
          console.log(newArr); 

          // [2, 4]

forEach: forEach method applies a function to each element of an array, but it does not create a new array. It simply iterates over the array and executes the provided function for each element.
Example: 

const arr = [1, 2, 3, 4, 5];
          arr.forEach((num) => console.log(num));
          
          // 1
          // 2
          // 3
          // 4
          // 5

find: find method returns the first element in an array that satisfies the specified condition in a function. If no elements satisfy the condition, it returns undefined.
Example: 

const arr = [1, 2, 3, 4, 5];
          const result = arr.find((num) => num > 3);
          console.log(result);

          // 4

map and filter both create new arrays, while forEach does not create a new array, and find returns the first element that satisfies the specified condition.

What is the difference in LocalStorage, SessionStorage, and cookies?

LocalStorage, SessionStorage, and cookies are ways that web developers can store information in a user's web browser using JavaScript.

Think of it like a little box where you can put things in and take them out later. Cookies are like a small box that you can only use for a little while. SessionStorage is like a bigger box that you can use for a longer time, but it goes away when you close your web browser. LocalStorage is like a big box that you can use for a really long time, even after you close your web browser.

So, if you want to store some information that you only need for a short time, like a username or a shopping cart, you can use cookies. If you want to store some information that you need for a longer time, like a user's preferences, you can use SessionStorage. If you want to store information that you need for a really long time, like a user's login information, you can use LocalStorage.

How does event loop work?

Imagine you're playing a video game where you have to collect coins and avoid obstacles. You press the arrow keys to move your character, and you can only move one step at a time. Now imagine that the video game has to handle other things too, like playing sound effects and keeping track of your score. All of these things are happening at the same time, but your character can only move one step at a time.

This is a bit like how JavaScript works. JavaScript is a programming language used to create websites and web applications. When you visit a website, the JavaScript code on that website is constantly running in the background, handling things like user interactions, animations, and network requests. The problem is that JavaScript can only do one thing at a time, just like your character in the video game can only move one step at a time. So how does JavaScript handle all of the different things that need to happen at the same time? Enter the Event Loop. The Event Loop is like the traffic cop of the JavaScript world. It makes sure that everything gets done in the right order, without anything getting stuck or delayed.

Here's how it works: Whenever an event happens in JavaScript (like a button click or a network request), the Event Loop adds that event to a queue. The queue is like a waiting line for events. Meanwhile, the JavaScript code continues to run, doing whatever it needs to do. When it's done, the Event Loop checks the queue to see if there are any events waiting to be processed. If there are, the Event Loop takes the first event in the queue and processes it. Once the event is done, the Event Loop goes back to the queue and takes the next event, and so on. This process continues indefinitely, with events being added to the queue and processed in the order they were added.

So, just like how you move your character one step at a time in the video game, JavaScript handles events one at a time using the Event Loop. This ensures that everything gets done in the right order, without anything getting stuck or delayed.

What does the function quizTimer do in noChange.js file?

The function quizTimer sets a timer that increments a count by 1 every second. It then displays the count in the format of minutes and seconds, and updates the HTML content of an element with the ID count. The covtMin variable calculates the minutes and the mod variable calculates the remaining seconds. The min and countSec variables add leading zeros to the minutes and seconds respectively, to ensure that they are always displayed in a consistent format.

The if (count > 60) condition checks if the timer has been running for more than 60 seconds. If it has, it changes the color of the text in the countHtml element every 2 seconds to make it more noticeable to the user. If the dismiss parameter is passed to the function as true, the clearInterval() function is called to stop the timer. This can be used to stop the timer when it is no longer needed.

The function chooseQuiz updates or adds an object to an array of quiz answers, and then displays the updated array of answers.

If there is no object in the answers array with the same id property as the quizData object at the specified index, the function creates a new object with the properties of the quizData object at the specified index, as well as the givenAns property, and pushes it to the answers array.
